Lesson 12

Line-by-Line Walkthrough: styles.css

This lesson breaks down the visual system of the project in a code-first way. It focuses on tokens, layout, cards, gradients, pills, chart styling, motion, and responsive behavior.

Lesson Contents

  • Design tokens
  • Page layout and shell
  • Cards and surfaces
  • Sidebar states

1. Design Tokens

:root {
  --bg: #0e1726;
  --surface: rgba(14, 25, 40, 0.72);
  --border: rgba(255, 255, 255, 0.09);
  --text: #edf4ff;
  --muted: #9fb3ca;
  --primary: #69b1ff;
  ...
}

This top block is the heart of the visual system. It defines the dashboard's language before any component appears. When you learn dashboard design, this is one of the most reusable lessons: if you control color, contrast, and surface tokens centrally, the rest of the system becomes much easier to theme and evolve.

2. Page Layout and Shell

.app-shell {
  min-height: 100vh;
  display: grid;
  grid-template-columns: 320px 1fr;
  gap: 24px;
  padding: 24px;
}

This is where the dashboard gets its high-level shape. The grid creates a left rail and a main working column. The design lesson here is that layout comes before decoration. The right background, card shadows, and gradients are only effective because the structural spacing is already clear.

3. Cards and Surfaces

.glass-card {
  background: var(--surface);
  border: 1px solid var(--border);
  box-shadow: var(--shadow);
  backdrop-filter: blur(20px);
}

The cards work because they combine three things:

  • a semi-transparent surface
  • a soft border
  • a shadow that separates the surface from the background

This is a very modern pattern because it makes the interface feel layered and atmospheric instead of flat.

4. Sidebar States and Collapsing

.app-shell.sidebar-collapsed {
  grid-template-columns: 92px 1fr;
}

.sidebar.is-collapsed .sidebar-copy h1,
.sidebar.is-collapsed .sidebar-copy .muted,
.sidebar.is-collapsed .sidebar-body {
  display: none;
}

This styling is a strong example of state-based CSS. Nothing magical happens here: JavaScript adds or removes classes, and CSS interprets those classes into a new layout. This separation of responsibilities is one of the cleanest ways to build interactive UI.

5. KPI Cards

.accent-blue  { background: linear-gradient(135deg, rgba(79, 155, 255, 0.18), var(--surface-strong)); }
.accent-gold  { background: linear-gradient(135deg, rgba(255, 175, 74, 0.18), var(--surface-strong)); }
.accent-coral { background: linear-gradient(135deg, rgba(255, 104, 111, 0.16), var(--surface-strong)); }
.accent-green { background: linear-gradient(135deg, rgba(65, 207, 147, 0.16), var(--surface-strong)); }

The KPI cards are a small masterclass in subtle emphasis. The gradients are not dramatic. They simply tint the upper-left portion of each card enough to create identity and depth. That is why they feel elegant instead of noisy.

6. Chart Styling

.chart-shell {
  position: relative;
  padding: 14px 16px 10px;
  border-radius: 24px;
  background: linear-gradient(180deg, rgba(255, 255, 255, 0.05), rgba(255, 255, 255, 0.02));
  border: 1px solid var(--border);
}

.chart-area-blur {
  fill: url(#chartAreaGradient);
  opacity: 0.9;
}

.chart-line {
  fill: none;
  stroke: var(--primary);
  stroke-width: 3;
}

This section teaches an advanced visual idea: a chart is not just data visualization. It is also a designed surface. The shell, the area fade, the glow, and the marker treatment are what make it feel integrated into the dashboard.

7. Motion and Transitions

.app-shell {
  transition: grid-template-columns 0.32s ease, gap 0.32s ease;
}

.sidebar-toggle {
  transition: transform 0.22s ease, background 0.22s ease, color 0.22s ease;
}

These transitions are short, readable, and intentional. They do not exist to show off. They exist to help the user feel the change in state. This is a major UI/UX lesson: motion should support comprehension.

8. Exercise and Checkpoint

Problem

Create one new accent class, such as .accent-purple, using the same gradient pattern as the existing KPI cards.

Solution Approach

Reuse the same card geometry and only change the accent tint. That keeps the dashboard cohesive while still adding variety.

.accent-purple {
  background: linear-gradient(
    135deg,
    rgba(174, 120, 255, 0.18),
    var(--surface-strong)
  );
}

Apply it in the HTML like this:

<article class="stat-card glass-card accent-purple">
  <p class="stat-label">Blocked</p>
  <strong class="stat-value">12</strong>
</article>

How It Should Look

The new card should feel like part of the same KPI family, but with a violet-tinted atmosphere. If it feels too loud, reduce the opacity in the gradient color.

Checkpoint: if you can explain the difference between token definitions, component styles, and state-based selectors, you understand the main architecture of styles.css.
