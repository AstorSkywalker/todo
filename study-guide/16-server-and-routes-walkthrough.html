Line-by-Line Walkthrough: server.js and routes

1. Imports and Mental Model

const http = require('http');
const fs = require('fs');
const path = require('path');
const { URL } = require('url');

The first thing to notice is that this server is intentionally lightweight. It does not use Express. That means every line teaches more about how Node's built-in HTTP server works.

const { handleTodosApi } = require('./routes/todos');
const { sendJson } = require('./utils/http');

These imports already tell a story:

  • handleTodosApi owns the todo routes
  • sendJson standardizes API responses
  • the server entry point delegates instead of doing everything itself

2. The Static File Map

const PUBLIC_FILES = {
  '/': 'index.html',
  '/docs': 'docs.html',
  '/study-guide': 'study-guide/index.html',
  '/openapi.json': 'openapi.json',
  '/styles.css': 'styles.css',
  '/app.js': 'app.js'
};

This object is a simple router for static assets. Instead of a framework abstraction, the app keeps a small explicit map. That makes it easy to teach and easy to inspect.

The design lesson here is clarity over cleverness. A small explicit object is often better than a complicated automatic system in a learning project.

3. Understanding createAppServer()

function createAppServer() {
  ensureDataFile();
  initializeSqlite();
  syncTodosToSqlite(readTodos());

  return http.createServer(async (req, res) => {
    const requestUrl = new URL(req.url, `http://${req.headers.host}`);
    ...
  });
}

This function does two jobs:

  1. prepare storage before serving requests
  2. return a server instance with request handling logic

That startup sequence is important. If the CSV file does not exist or SQLite is not initialized, the server could start in a broken state. Good startup code prevents later surprises.

if (requestUrl.pathname.startsWith('/api/todos')) {
  await handleTodosApi(req, res, requestUrl);
  return;
}

This is the API gate. Any request beginning with /api/todos leaves the bootstrap file and enters the route layer.

4. How serveStaticFile() Works

const filePath = path.join(__dirname, fileName);
const normalizedPath = path.normalize(filePath);
const workspacePath = path.normalize(__dirname + path.sep);

This part is about safety. The server normalizes paths before reading files so a malformed request cannot wander outside the project directory.

const contentType = {
  '.html': 'text/html; charset=utf-8',
  '.css': 'text/css; charset=utf-8',
  '.js': 'application/javascript; charset=utf-8',
  '.json': 'application/json; charset=utf-8',
  '.md': 'text/markdown; charset=utf-8'
}[extension] || 'text/plain; charset=utf-8';

This mapping is one of those small but important details. Browsers need the right content type to interpret files correctly.

5. Reading routes/todos.js

if (req.method === 'GET' && pathParts[2] === 'storage') {
  sendJson(res, 200, getStorageStatus());
  return;
}

The route file is organized as a sequence of method/path checks. Each block answers a very specific question:

  • Is this an export request?
  • Is this a storage status request?
  • Is this an import request?
  • Is this a list request?
  • Is this a group request?
  • Is this an item-by-id request?
const result = createTodo(body);

if (result.error) {
  sendJson(res, result.statusCode, { error: result.error });
  return;
}

sendJson(res, result.statusCode, result.item);

This pattern appears repeatedly and it is a good one to learn: route handlers ask services for a result object, then translate that object into an HTTP response.

6. Important Backend Patterns To Notice

PatternWhat it meansWhy it helps
Early returnsEach route branch ends quickly once matched.Keeps control flow readable.
Service result objectsServices return data plus status or error info.Keeps routes thin and predictable.
Shared helpersreadBody and sendJson are reused.Reduces repetition.
Startup orchestrationInitialization happens before request handling.Makes the app safer to boot.

7. Exercise and Checkpoint

Problem

Add a tiny static route so visiting /ping.txt returns a plain text response saying TaskFlow is alive.

Solution Approach

The quickest way is to intercept that path in createAppServer() before the static file mapping.

if (requestUrl.pathname === '/ping.txt') {
  res.writeHead(200, { 'Content-Type': 'text/plain; charset=utf-8' });
  res.end('TaskFlow is alive');
  return;
}

How It Should Look

Opening /ping.txt in the browser should show a simple line of plain text. That confirms you understand how requests are branched before the rest of the routing system runs.

Checkpoint: if you can explain how requests move from server.js into routes/todos.js and back out as JSON, you understand the outer shell of the backend.
