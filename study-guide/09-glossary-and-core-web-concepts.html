Lesson 9

Glossary and Core Web Concepts

This lesson is a beginner-friendly reference section. It explains the most important words and ideas used across the project so you can study the other lessons with less friction.

Lesson Contents

  • HTML concepts
  • CSS concepts
  • JavaScript concepts
  • Backend and API terms
  • Data and storage terms
  • Documentation and testing terms

1. HTML Concepts

TermMeaningHow it appears in this project
elementA piece of HTML such as <div> or <button>.The dashboard is built from many semantic elements.
attributeExtra information placed inside an element tag.id, class, type, and aria-label are used heavily.
semantic HTMLChoosing elements based on meaning, not just appearance.<aside>, <main>, <section>, and <article> help structure the dashboard.
templateHidden HTML structure that JavaScript can clone later.The task cards and compact rows use <template> blocks.
formA group of inputs submitted together.The task creator/editor uses one main form.

2. CSS Concepts

TermMeaningHow it appears in this project
selectorThe pattern that chooses what to style..glass-card, .stat-card, and body.light are selectors.
custom propertyA CSS variable like --primary.All themes and palettes depend on these variables.
gridA layout system for rows and columns.The page shell, stats row, filter layout, and compact task rows all use grid.
flexboxA layout system for one-dimensional alignment.Buttons, headings, toolbars, and pills use flexbox.
transitionA smooth animation between CSS values.The sidebar, chevrons, buttons, and theme toggle all use transitions.
gradientA blended color background.The KPI summary cards use subtle accent gradients.

3. JavaScript Concepts

TermMeaningHow it appears in this project
stateThe current data that the UI depends on.The state object stores view mode, sort mode, grouping mode, and rendered items.
event listenerA function that runs when the user interacts with something.Buttons, form submissions, filters, and toggles all use listeners.
renderingTurning data into visible UI.Task cards, group results, summary counters, and the chart are all rendered from data.
localStorageBrowser storage for small persistent values.Theme, palette, collapsed states, filters, and view preferences are saved here.
fetchA browser API for making HTTP requests.The frontend uses it to call the backend endpoints.

4. Backend and API Terms

TermMeaningHow it appears in this project
routeA path and HTTP method handled by the server.GET /api/todos and POST /api/todos are routes.
CRUDCreate, Read, Update, Delete.The task API supports all four operations.
JSONA text format for structured data.The API returns JSON responses and accepts JSON payloads.
serviceA module that holds business logic.todoService.js contains the main storage and task operations.
validationChecking whether input is acceptable.The backend rejects invalid dates, missing titles, or malformed import data.

5. Data and Storage Terms

TermMeaningHow it appears in this project
CSVComma-separated values stored as text rows.data/todos.csv is the source of truth.
SQLiteA lightweight relational database stored in one file.data/todos.db mirrors the CSV data.
source of truthThe storage layer the app trusts most.Right now that is still the CSV file.
syncKeeping two storage systems aligned.Server startup and write operations keep CSV and SQLite aligned.

6. Documentation and Testing Terms

TermMeaningHow it appears in this project
OpenAPIA standard machine-readable API description.The project stores it in openapi.json.
Swagger UIA browser interface that reads OpenAPI and lets you test endpoints.Available at /docs.
test suiteA set of automated checks.scripts/run-tests.js covers CRUD, storage sync, import/export, grouping, and errors.
smoke testA quick test that checks whether the main system still basically works.A future frontend testing idea for this project.

Study tip: when a lesson uses a word you do not fully know, come back to this glossary. That is faster than trying to guess what the code means from context alone.
