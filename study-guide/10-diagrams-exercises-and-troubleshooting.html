2. Request and Persistence Maps

Here is a more detailed request flow for a read operation such as GET /api/todos:

Browser
  -> fetch('/api/todos')
Server bootstrap
  -> routes/todos.js
Route handler
  -> todoService.listTodos()
Service
  -> csvService.readTodosFromCsv()
Service
  -> apply search/filter/sort/group logic
Route
  -> sendJson(res, 200, todos)

And here is the write flow for a create or update request:

Browser form submit
  -> JSON payload
Route handler
  -> readBody(req)
  -> validation helpers
Service layer
  -> read existing CSV rows
  -> mutate in memory
  -> write CSV file
  -> sync SQLite mirror
Route
  -> return updated JSON response

One more important map is local browser persistence:

localStorage
  -> theme
  -> color scheme
  -> filters
  -> sort mode
  -> compact/card view
  -> density mode
  -> collapsed panel states
Frontend initialize()
  -> restore preferences
  -> apply UI state
  -> load tasks
  -> render dashboard

These diagrams matter because a learner often understands code much faster after first understanding the shape of the system.

3. Rebuild Roadmap

If you want to recreate the app from zero instead of only reading it, use this structured path:

  1. Create a static page shell with sidebar and dashboard regions.
  2. Add a small form and static cards with fake sample text.
  3. Style the cards and layout system.
  4. Add frontend JavaScript state and task rendering.
  5. Add a tiny backend that serves the static files.
  6. Add a basic GET /api/todos route.
  7. Add CSV persistence.
  8. Add create, update, and delete.
  9. Add theme switching and palettes.
  10. Add grouping, filters, and compact view.
  11. Add the chart.
  12. Add import/export, tests, and docs.

Teaching reminder: do not try to rebuild everything at once. Rebuild one vertical slice at a time, then compare your version to the real project.

5. Troubleshooting Guide

ProblemLikely causeWhat to try
The UI changed but I still see the old version.The browser cached something or the page was not refreshed.Refresh the page, or do a hard refresh if needed.
A new backend route does not work.The server is still running old code.Restart the Node server after backend changes.
The date format looks wrong.Old localStorage values or inconsistent formatting.Refresh after the latest code, or clear the relevant stored preference.
The chart looks broken after resizing or style changes.The CSS aspect ratio and SVG coordinate system are out of sync.Check the SVG viewBox, chart width/height constants, and the dot overlay sizing together.
The study guide links feel broken.Relative paths or server routes are wrong.Use explicit routes like /study-guide/... and make sure the server serves the files.
Theme or collapse state behaves unexpectedly.A saved localStorage value may be older than the latest implementation.Refresh and, if needed, clear the related localStorage entry.
